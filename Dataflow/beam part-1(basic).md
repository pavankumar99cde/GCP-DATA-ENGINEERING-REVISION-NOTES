## Dataflow Terminology quick refer

- **Apache Beam** :- Framework used to build batch and streaming pipelines.
- **Dataflow** :- Google Cloud service that runs Beam pipelines.
- **Pipeline** :- Complete data processing workflow from source to destination.
- **PipelineOptions** :- Configuration values passed to the pipeline.
- **Runner** :- Engine that executes the Beam code.
- **DataflowRunner** :- Runs pipeline on Google Cloud Dataflow.
- **DirectRunner** :- Runs pipeline locally for testing.
- **PCollection** :- Distributed dataset flowing through the pipeline.
- **Bounded PCollection** :- Finite data (CSV, Database Table, File).
- **Unbounded PCollection** :- Infinite streaming data (Pub/Sub, Kafka).
- **PTransform** :- Processing step applied on a PCollection.
- **ParDo** :- Most common transformation; processes each record independently.
- **DoFn** :- Custom business logic used inside ParDo.
- **ProcessElement** :- Method where record-level processing happens.
- **Map** :- One input record → One output record.
- **FlatMap** :- One input record → Multiple output records.
- **Filter** :- Removes unwanted records.
- **GroupByKey** :- Groups records having same key.
- **CombinePerKey** :- Aggregates values for each key.
- **CoGroupByKey** :- Joins multiple PCollections using key.
- **Partition** :- Splits data into multiple outputs.
- **Flatten** :- Merges multiple PCollections into one.
- **Read** :- Reads data from source.
- **Write** :- Writes data to destination.
- **Source** :- Place where data comes from.
- **Sink** :- Place where data is stored.
- **Kafka** :- Streaming platform often used as source.
- **Schema** :- Structure of the data.
- **KV** :- Key-Value pair.
- **Tuple** :- Collection of multiple values.
- **TupleTag** :- Used for multiple outputs in ParDo.
- **Side Output** :- Additional output generated by ParDo.
- **Main Output** :- Primary output generated by ParDo.
- **Side Input** :- Small lookup dataset passed to ParDo.
- **View** :- Converts PCollection into side input format.

- **Trigger** :- Decides when results should be emitted.
- **Watermark** :- Beam's estimate of event completeness.
- **Event Time** :- Time when event actually happened.
- **Processing Time** :- Time when Beam processes event.
- **Ingestion Time** :- Time when event entered system.
- **Late Data** :- Event arriving after expected time.
- **Allowed Lateness** :- Time allowed for late records.

  
- **State** :- Stores intermediate data for processing.
- **Timer** :- Executes logic after specific time.
- **Aggregation** :- Summarizing multiple records.
- **CombineFn** :- Custom aggregation logic.
- **WindowFn** :- Custom windowing logic.
- **Serialization** :- Converting object into transferable format.
- **Coder** :- Beam serializer/deserializer.
- **Row** :- Schema-aware Beam record.
- **Beam SQL** :- SQL queries on PCollections.
- **Metrics** :- Custom counters and monitoring.
- **Counter** :- Counts occurrences.
- **Distribution** :- Tracks min, max, average values.
- **Gauge** :- Tracks latest value.
- **Worker** :- VM processing Beam tasks.
- **Worker Pool** :- Group of workers running the pipeline.
- **Autoscaling** :- Automatically adds/removes workers.
- **Job** :- Running pipeline instance.
- **DAG** :- Directed graph of transformations.
- **Shuffle** :- Data redistribution across workers.
- **Fusion** :- Beam optimization combining transforms.
- **UDF** :- User Defined Function.
- **Template** :- Reusable Dataflow job definition.
- **Flex Template** :- Docker-based reusable Dataflow template.

| Feature              | Fixed Window (Tumbling)                      | Sliding Window (Hopping)                   | Session Window                                                   | Global Window                            |
| -------------------- | -------------------------------------------- | ------------------------------------------ | ---------------------------------------------------------------- | ---------------------------------------- |
| **Definition**       | Fixed-size, non-overlapping time intervals   | Fixed-size windows that overlap            | Groups events separated by inactivity gaps                       | Entire dataset belongs to one window     |
| **Window Boundary**  | Predefined start and end times               | Predefined size + slide interval           | Dynamically determined by activity                               | No practical boundary                    |
| **Overlap**          | No                                           | Yes                                        | No                                                               | N/A                                      |
| **Window Creation**  | Created at fixed intervals                   | Created periodically based on slide period | Created when activity starts and merged while activity continues | Single window for all elements           |
| **Event Assignment** | Event belongs to exactly one window          | Event can belong to multiple windows       | Event belongs to a user/session-specific window                  | Every event belongs to the same window   |
| **Based On**         | Time                                         | Time                                       | Activity gap                                                     | Entire stream                            |
| **Example**          | Every 5 minutes                              | Window size = 10 min, slide every 5 min    | User inactive for 30 min ends session                            | All events from stream start             |
| **Best Use Case**    | Hourly sales, periodic reporting             | Running averages, trend analysis           | User behavior, clickstream analysis                              | Custom triggering, lifetime aggregations |
| **Output Frequency** | Once per window (unless triggers configured) | Multiple overlapping outputs               | When session closes or trigger fires                             | Depends entirely on triggers             |
| **Dataflow Name**    | Fixed Window                                 | Sliding Window (Hopping Window)            | Session Window                                                   | Global Window                            |
